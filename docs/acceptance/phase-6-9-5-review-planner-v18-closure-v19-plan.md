# Phase 6.9.5 Review / Planner V18 Closure and V19 Plan

Date: 2026-07-20

V18's one branch product command returned fixed `default_off` before owner,
ledger, Docker, browser, API, provider or synthetic resources. Its public,
recovery and execution roots remain absent, so V18 is immutable, non-retryable
and not recovery-admissible.

The command's confirmation arguments were verified as the expected two values;
the cause is therefore not Bun separator forwarding. A direct Bun host
preflight returned `ready`, while the Node-runner product path still closed at
preflight. That host-runtime parity difference must be proven without spending
another product lineage.

V19 is the independent replacement lineage. It has its own confirmation,
schemas, ledger, recovery, execution, browser and public-evidence namespaces.
It retains the repository-root CWD, strict allowlists, read-only Bun V10
authority bridge, receipt/default-off/attempt/recovery rules and V10--V18
sentinels.

Before V19 product acceptance, the new read-only `preflight:review-planner:v19:product`
command runs the exact V19 Node runner, strict product confirmation parser and
default host preflight, but stops before owner acquisition and cannot create a
ledger, Docker mutation, browser, API, provider or synthetic account. Only a
ready result permits the separately authorized one-shot V19 product command.

V10 remains the sole semantic-quality authority and both business gates remain
`false`.
