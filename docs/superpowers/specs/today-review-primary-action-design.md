# 今日复习主操作同页反馈设计

## 目标

让 `/today` 页面 Review/Planner 建议卡的“先完成今日复习”成为可感知、可访问的操作，而不是跳回当前 URL 后没有任何反馈的链接。

## 已确认的根因

建议卡从首个 Planner block 取得 `targetHref`，并统一渲染为 Next `Link`。当它在 `/today` 内的目标仍是 `/today` 时，客户端导航没有页面或焦点变化，因此用户看不到点击结果。

## 方案选择

采用本地操作回调，而不是 URL 锚点或强制刷新：

- 建议卡新增可选的 `onPrimaryAction`；存在时优先触发该回调，否则保持现有受控 `Link` 导航。
- 仅 `/today` 的 compact 建议卡传入该回调；`/plan` 和其他调用方继续导航至 `/today`。
- 回调将平滑滚动至“今日复习”区域，聚焦第一张待评分任务；只有任务请求已成功结束且无待评分任务时，才聚焦该区域并通过既有 notice 明确告知用户。
- 目标区域使用稳定 DOM id 与可编程焦点，不引入全局路由状态、数据写入或新的 API。

## 边界与无障碍

- 该操作只改变当前页面的视图焦点，不会评分、跳过、创建或修改任何 `ReviewTask`。
- 目标元素须在键盘/辅助技术中可获得焦点；焦点移动后页面阅读位置与视觉位置一致。
- 平滑滚动不可用时，浏览器默认滚动仍可到达目标。
- suggestion 尚未加载、建议无首个 block、或待复习任务仍在加载/读取失败时，卡片原有渲染与加载边界不变，不能把未返回数据误报为空状态。

## 验收

1. 针对同页 `onPrimaryAction` 先写一个会失败的 UI 集成测试：点击不再产生 `/today` 链接导航，而是调用传入的本地操作。
2. `/plan` 未传回调时仍保持安全的 `/today` 链接；不可信 href 仍标准化至 `/today`。
3. `/today` 的回调会选择第一个 `PENDING` 任务作为焦点目标；空待办时选择复习区域并给出已有 notice。
4. 执行受影响 Web 测试、类型检查/构建，并在可见浏览器中验证 `/today` 的滚动、焦点和空态反馈。
5. 同步 DEVLOG 与必要的验收记录；分支审查通过后合并 main，执行 main 回放并推送。
